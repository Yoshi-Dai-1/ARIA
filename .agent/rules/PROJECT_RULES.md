# ARIA プロジェクト共通ルール (Project Rules)

これらのルールは、エージェント（AI）が Aria プロジェクトで行動する際の「憲法」であり、あらゆるスキルやワークフローよりも優先される。

## 1. コミュニケーションと言語
- **日本語の優先**: ユーザーへの回答、コード内のコメント、およびドキュメントは原則として日本語で行う。技術用語には英語を併記する。
- **透明性の確保**: 重要な変更（ファイルの分割、ロジックの抜本的修正）を行う前には、必ず技術的な理由を添えてユーザーに報告し、承認を得る。

## 2. 開発・エンジニアリング
- **現状維持と漸進的改善**: 既存の資産を尊重し、一度に大規模なリファクタリングを独断で行わない。
- **フェイルファスト (Fail-Fast)**: データ整合性に疑義がある場合、処理を強行せず直ちに停止し、ユーザーに状況を共有する。
- **NULL 基底の遵守**: 不明なデータは適当な値で埋めず、NULL (None) として扱う。

## 3. プロジェクトビジョンと持続可能性 (Sustainability)
ARIA プロジェクトは、以下の条件を永久に満たすことを目指す。
- **完全無料での永久稼働**: GitHub Actions による自動実行と Hugging Face によるデータストレージを基盤とし、インフラコストを 0 に抑える。
- **最小限のメンテナンス**: 外部仕様変更に強い疎結合な設計を維持し、数年放置しても動作し続ける持続可能性を追求する。
- **高セキュリティと安定性**: 投資アプリとして、データの機密保持と計算の正確性を絶対条件とする。

## 4. 知能の自律監視・改善義務 (Autonomous Management)
エージェント（AI）は、プロジェクトの進化に伴い、以下の行動を自律的に行う義務を負う。
- **適宜見直し**: Rules, Skills, Workflows について、追加・統合・分離・削除・修正の必要性を常に判断する。
- **報告の義務**: 修正の必要性を検知した場合、独断で行わず、必ずユーザーにその旨と理由を報告し、承認を得る。
- **基準の遵守**: 修正に際しては、[antigravity.google](https://antigravity.google/docs/rules-workflows) および [agentskills.io](https://agentskills.io/home) の仕様を熟読し、それに従う。
