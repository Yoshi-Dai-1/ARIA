# セキュリティ・ガイドライン (Security Guidelines)

投資アプリという機密性の高いデータを扱う特性上、ARIA では以下のセキュリティ基準を遵守する。

## 1. 秘密情報管理 (Secrets Management)
- [ ] **環境変数の徹底**: APIキー、トークン、パスワードをコード内にハードコードしていないか。
- [ ] **ログの安全性**: ログ出力に秘密情報や PII（個人を特定できる情報）が含まれていないか。
- [ ] **.gitignore**: `.env` や認証ファイルが Git 管理から除外されているか。

## 2. データアクセスの最小化
- [ ] **書き込み制限**: ストレージへの書き込みが、必要なフォルダ・ファイルのみに制限されているか。
- [ ] **API 呼び出しの妥当性**: 不要な外部通信が発生していないか。

## 3. 依存関係の健全性
- [ ] **パッケージ監査**: 怪しい野良ライブラリや、脆弱性が報告されているバージョンを使用していないか。
- [ ] **サブモジュールの保護**: 外部から提供されたライブラリ（edinet_xbrl_prep 等）を改ざんしていないか。

## 4. 入力バリデーション
- [ ] **不純物の排除**: 外部（API, CSV）からのデータ投入時に、型・形式の妥当性チェックが行われているか。
- [ ] **異常値のトラップ**: 異常な巨大数値や特殊文字によるインジェクションの防止。
