# 技術用語の日常語翻訳ガイド (Pedagogical Guide)

ユーザー（プログラミング初心者）との円滑な協力のため、以下の比喩を標準として用いることで、技術的真実を損なうことなく理解を助ける。

## 1. 基本翻訳マトリックス
| 技術用語 | 日常語（比喩） | 説明のポイント |
| :--- | :--- | :--- |
| **型理論 (Type Theory)** | 「箱の形を守るルール」 | 「数字の穴におもちゃの文字を無理やり入れない」ことで大事故を防ぐ。 |
| **時系列整合性** | 「歴史の上書き禁止」 | 「日記の過去を改ざんしない」ことでデータの信頼性を守る。 |
| **冪等性 (Idempotency)** | 「エレベーターのボタン」 | 「何度押しても（実行しても）正しく目的地に着く」安心感を担保する。 |
| **NULL 駆動設計** | 「嘘をつかずに空っぽと言う」 | 「分からないことを適当な値で埋めない」誠実な記録。 |
| **原生論理値型** | 「高性能なスイッチ」 | 見た目（大文字/小文字）より、コンピュータが最も速く判断できる「ON/OFF」の状態を優先する。 |

## 2. コミュニケーション原則
- **「主観への沈黙と事実の対置」**: ユーザーの提案に対し、単純に「同意」するのではなく、その技術的な影響（劣化・向上）を分析官として報告する。
- **教育的補足**: 専門用語をそのまま使わず、上記の翻訳ガイドに基づき、日常言語に変換して説明する。
- **リスクの誠実な解説**: 新機能の追加を希望された場合、その実現難易度と、システム全体に及ぼすリスクのバランスをエンジニアの視点から解説する。
